Images (SVG)

Internet Explorer 8 is the last browser that doesn't support SVG images and backgrounds, so unless support is required, there is no need for PNGs as various sizes. The other advantage of using SVGs is the auto-scaling for high DPI displays (e.g. retina).

If support is required, the following trick can be used for background images to fallback to PNGs:

.harvest-wheat {
    background: transparent url(img/wheat-128.png) no-repeat;
    background-image: url(img/wheat.svg), none;
    background-size: contain;
    height: 128px;
    width: 128px;
}

This uses the PNG by default and then specifies a background-image with the SVG and a second empty one. Browsers which support SVG also support multiple background images. IE8 will not recognize the multiple background image syntax and stick with the PNG.
